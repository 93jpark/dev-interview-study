## AJAX란?

---

> Asynchronous JavaScript and XML 의 약자로 XMLHttpRequset 객체이용함으로서
서버와 브라우저가 비동기적인 방식으로 데이터를 통신합니다.

##  AJAX를 사용하는 이유

---

![Ajax-vergleich-en](https://user-images.githubusercontent.com/80400157/205499277-7386f6ce-15ad-42aa-a556-787619f1a207.svg)
#### 기존의 웹 애플리케이션 과정
- FORM을 채우고 SUBMIT을 통해 웹서버에 요청
- 요청에 따른 데이터를 가공하여 새로운 웹페이지를 작성 후 응답
- 하지만 일부분을 제외하고 대부분 유사한 HTML코드를 다시 전송받음 
- 대역폭 낭비 및 금전적 손실

#### AJAX 어플리케이션 과정
- 필요한 데이터만을 웹서버에 요청
- 클라이언트에서 데이터에 대한 처리 (기존에 웹서버에서 전적으로 처리되는걸 일부 클라이언트에서 처리)
- 브라우저와 서버 사이에 교환되는 데이터가 줄어듬
- 데이터 처리량이 줄고 애플리케이션의 응답성이 좋아짐

>  데이터를 조회할때 페이지 전체가 새로고침 되는것이 아닌 JSON, XML 형태로 필요한 데이터만을
갱신하기에 대역폭낭비와 처리량을 줄이고 응답속도를 높일수있다

## AJAX 장/단점
---

#### 장점
- 앞서 말한대로 웹페이지 전체가 아닌 일부분만을 갱신할수있음
- 웹페이지 로드후 서버에 요청및 데이터를 보내고 받을수 있음
- 서버의 처리를 비동기적으로 처리할수있음

#### 단점
- Ajax 스크립트가 포함된 서버가 아닌 다른 서버로 Ajax 요청을 보낼 수는 없음
- 히스토리 관리가 안되어 보안에 신경써야한다
- 연속으로 요청을 할경우 부하가 증가할수 있다.
